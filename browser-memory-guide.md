# 브라우저 메모리 절약 가이드 (Edge / Chrome)

Edge와 Chrome은 둘 다 Chromium 기반이라 설정 항목이 거의 같다. 메뉴 경로와 명칭만 다르다.

16GB 환경에서 브라우저는 보통 IDE 다음으로 메모리를 많이 먹는다. 원인은 대부분 **탭 개수**와 **확장 프로그램**이고, 설정 조정은 그다음이다. 아래는 효과가 큰 순서대로 정리했다.

---

## 0. 먼저 현황 확인 — 브라우저 작업 관리자

`Shift + Esc` (Edge/Chrome 공통). 탭·확장·GPU 프로세스별 메모리 사용량이 나온다. 메모리 열을 기준으로 정렬하면 범인이 바로 보인다.

여기서 확장 프로그램 하나가 수백 MB를 먹고 있는 경우가 흔하다. 설정을 만지기 전에 이걸 먼저 본다.

---

## 1. 메모리 세이버 / 절전 모드 (효과 가장 큼)

일정 시간 안 쓴 탭을 메모리에서 내리는 기능. 탭은 남아 있고 다시 누르면 새로고침된다.

### Chrome

`chrome://settings/performance` → **메모리 절약 모드** 켜기

- 모드는 3단계(보통/균형/최대)가 있다. 16GB면 **최대**가 무난하다.
- **항상 활성 상태로 유지할 사이트**에 슬랙·구글 문서·로컬 개발 서버(`localhost`) 등 비활성화되면 곤란한 사이트를 등록한다.

### Edge

`edge://settings/system` → **리소스** 섹션

- **절전 모드로 리소스 절약** 켜기
- **절전 모드 탭** (Sleeping tabs) 켜기 + **비활성 상태 시간**을 기본 2시간에서 **5분~30분**으로 줄인다. 이 타이머 단축이 체감 효과가 가장 크다.
- **다음 사이트를 절대 절전 모드로 전환하지 않음**에 예외 사이트 등록.

> 트레이드오프: 절전된 탭을 다시 누르면 페이지가 다시 로드된다. 작성 중이던 폼 입력이 날아갈 수 있는 사이트는 반드시 예외에 넣는다.

---

## 2. 확장 프로그램 정리

확장은 대부분 백그라운드에서 상주하며, 일부는 탭마다 스크립트를 주입해 탭 개수에 비례해 메모리를 먹는다.

- Chrome: `chrome://extensions` / Edge: `edge://extensions`
- 안 쓰는 것은 **삭제**한다. "사용 중지"만으로도 메모리는 회수되지만, 목록에 남아 있으면 결국 다시 켜게 된다.
- 남길 확장은 **사이트 액세스**를 `클릭할 때`로 제한한다. (확장 상세 → 사이트 액세스) 모든 사이트에서 상시 동작하지 않게 된다.

---

## 3. 브라우저를 닫아도 백그라운드 앱 계속 실행 — 끄기

창을 다 닫아도 트레이에 남아 프로세스를 유지하는 옵션이다. 알림이 필요 없으면 끈다.

- Chrome: `chrome://settings/system` → **Google Chrome을 닫아도 백그라운드 앱 계속 실행** 끄기
- Edge: `edge://settings/system` → **Microsoft Edge가 닫혀 있을 때 백그라운드 확장 및 앱을 계속 실행** 끄기

---

## 4. 시작 부스트 (Startup boost) — 끄기

부팅 시 브라우저 프로세스를 미리 띄워 두는 기능. 실행 속도는 빨라지지만 안 쓰는 동안에도 메모리를 점유한다.

- Edge: `edge://settings/system` → **시작 부스트** 끄기
- Chrome에도 동일 항목이 있는 버전이 있다. `chrome://settings/system`에서 보이면 끈다.

setup-guide 12번(시작 프로그램 정리)과 같은 맥락이다.

---

## 5. 페이지 미리 로드 (Preload) — 끄기

다음에 방문할 것 같은 페이지를 미리 받아 두는 기능. 메모리와 네트워크를 함께 쓴다.

- Chrome: `chrome://settings/performance` → **더 빠른 검색 및 탐색을 위한 페이지 미리 로드** 끄기
- Edge: `edge://settings/privacy` → **더 빠른 검색 및 탐색을 위해 페이지 미리 로드** 끄기

> 트레이드오프: 체감 로딩 속도가 조금 느려진다. 메모리 여유가 아쉬울 때만 끈다.

---

## 6. Edge 전용 — 부가 기능 정리

Edge는 기본 활성화된 부가 기능이 많고, 각각이 별도 프로세스를 띄운다.

- **Copilot / 사이드바**: `edge://settings/sidebar` → 사이드바 표시 끄기, 개별 앱 끄기
- **쇼핑·쿠폰 기능**: `edge://settings/privacy` → 쇼핑 관련 항목 끄기
- **시작 페이지 피드**: 새 탭 페이지 우측 상단 톱니 → **콘텐츠 끄기**. 뉴스 피드가 새 탭마다 로드되지 않는다.

---

## 7. 하드웨어 가속은 켜 둔다

메모리 절약 목적으로 하드웨어 가속을 끄는 조언이 돌아다니는데, 대개 역효과다. 끄면 렌더링이 CPU/시스템 RAM으로 넘어와 **CPU 사용률과 발열이 올라간다**. Core Ultra 5 125H의 내장 GPU는 충분하므로 켜 둔다.

- 화면 깨짐·깜빡임 같은 렌더링 이상이 있을 때만 끄고 테스트한다.
- Chrome: `chrome://settings/system` / Edge: `edge://settings/system`

---

## 8. 프로세스 분리 완화 (선택, 고급)

Chromium은 보안을 위해 사이트마다 프로세스를 분리한다. 프로세스 하나당 고정 오버헤드가 있어 탭이 많으면 이게 누적된다. 바로 가기 대상에 플래그를 추가해 같은 사이트의 탭들을 한 프로세스로 묶을 수 있다.

바로 가기 우클릭 → 속성 → **대상** 끝에 한 칸 띄우고 추가:

```
--process-per-site
```

> 트레이드오프: **사이트 격리(Site Isolation)가 약해져 보안이 내려가고**, 한 탭이 죽으면 같은 사이트의 다른 탭도 함께 죽는다. 탭을 수십 개씩 열어야 하는데 메모리가 정말 모자랄 때만 쓴다. 기본은 권장하지 않는다.

---

## 9. 습관 — 설정보다 효과가 크다

- **탭 개수 관리**: 탭 30개는 어떤 설정으로도 감당이 안 된다. 나중에 볼 것은 즐겨찾기나 읽기 목록으로 내린다.
- **탭 그룹 접기**: 그룹을 접어 두면 메모리 세이버가 해당 탭들을 더 적극적으로 내린다.
- **브라우저 하나만 쓰기**: Edge와 Chrome을 동시에 띄워 두면 각각 기본 프로세스 세트를 갖는다. 개발용 프로필 분리는 브라우저를 나누는 대신 **같은 브라우저의 프로필 기능**으로 처리한다.
- **주기적 재시작**: 장시간 켜 두면 메모리가 계속 늘어난다. 하루 한 번 재시작이 가장 확실하다.

---

## 요약 체크리스트

| 항목 | Chrome 경로 | Edge 경로 | 설정 |
|---|---|---|---|
| 메모리 세이버 | `chrome://settings/performance` | `edge://settings/system` | 켜기 (Edge는 타이머 단축) |
| 확장 정리 | `chrome://extensions` | `edge://extensions` | 삭제 + 사이트 액세스 제한 |
| 백그라운드 실행 | `chrome://settings/system` | `edge://settings/system` | 끄기 |
| 시작 부스트 | `chrome://settings/system` | `edge://settings/system` | 끄기 |
| 페이지 미리 로드 | `chrome://settings/performance` | `edge://settings/privacy` | 끄기 |
| 하드웨어 가속 | `chrome://settings/system` | `edge://settings/system` | **켜 두기** |
